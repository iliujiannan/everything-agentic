# 阶段 4：单元测试

## 前置条件

- 已接收：阶段 3 完成的功能代码文件列表
- 代码已通过编译，无 CRITICAL/HIGH 代码审查问题

## 执行步骤

1. **分析覆盖缺口** — 运行现有测试，查看覆盖率报告，定位未覆盖的核心逻辑
2. **编写测试** — 调用 `/tdd` 命令，遵循 AAA 模式（Arrange-Act-Assert）：
   - 优先覆盖核心业务逻辑、边界条件、异常路径
   - 集成测试使用 Testcontainers（或项目约定的方式）
3. **验证覆盖率** — 调用 `/test-coverage` 确认整体覆盖率 ≥ 80%
4. **修复失败用例** — 修复 implementation 而非测试（除非测试本身有误）

## 产出物

- 测试代码文件
- 覆盖率报告摘要

## Gate 条件

- 覆盖率 ≥ 80%（不满足时停留在本阶段，不得进入 Sonar）

## 阶段摘要

完成后输出：

```
## 阶段摘要：单元测试
- 状态：PASS / FAIL
- 产出物：{测试文件列表}
- Gate 条件：覆盖率 {X}%（≥80% 为 PASS）
- 待确认事项：输入 [继续] 进入 Sonar 扫描阶段
```

**输出摘要后停止，等待用户 [继续]。**
